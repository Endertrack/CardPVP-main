# CardPVP — 游戏规则手册

## 游戏概述

**类型**：回合制卡牌对战  
**目标**：合理利用卡牌将对手 HP 降至 0  
**人数**：推荐 2 人

---

## 基础规则

### 血量与卡牌上限
- 默认血量上限：**20**
- 默认手牌上限：**10**
- 当拥有的手牌数达到卡牌上限时，新获得的手牌应立即丢弃并触发丢弃相关事件

### 开局流程
1. 随机决定先手
2. 双方各摸 **2** 张牌
3. 先手方开始第一个回合

### 回合流程
1. **回合摸牌**：轮到自己的回合时，先摸 **3** 张牌
2. **出牌阶段**：可以使用手牌，不限制出牌次数
3. **结束回合**：不想继续出牌时可以结束回合，轮到对手

### 数值规则
- 造成伤害或回血的效果量为**非正数时视为 0**，但仍算作 1 次伤害或回血事件，仍触发相关效果事件
- 向上取整：若数值存在小数部分，则向上取整数
- 向下取整：若数值存在小数部分，则向下取整数

## 卡牌系统

### 卡牌结构
每张卡牌包含：
- **名称**：卡牌标识
- **图标**：对应 `assets/item/{id}.png`（或 `.gif`）
- **消耗类型**：决定使用限制
- **事件**：卡牌效果（效果种类、数值、持续回合数等）
- **权重**：牌组中该卡的出现频率

### 消耗类型

| 标识 | 名称 | 限制 |
|------|------|------|
| icon1 | **行动卡** | 每回合只能出 **1** 张 |
| icon2 | **锦囊卡** | 每回合最多出 **3** 张 |
| icon3 | **回血卡** | 无限制 |
| icon4 | **攻击卡** | 无限制 |
| icon5 | **增益卡** | 无限制 |
| icon6 | **减益卡** | 无限制 |
| icon7 | **事件卡** | 无限制 |
| icon8 | **装备卡** | 占用装备槽，同类型替换 |
| icon9 | **武器卡** | 占用武器槽，同类型替换 |
| icon10 | **场地卡** | 占用场地槽，同类型替换 |
| icon11 | **策略卡** | 用于应对其它卡，在目标卡打出后打出 |

#### 装备/武器/场地替换规则
打出 icon8/icon9/icon10 卡牌时，若我方已有同类型卡生效：
1. 丢弃生效的同类型卡（触发丢弃事件）
2. 新打出的卡立即生效
3. 旧卡产生的 buff 被移除

## 效果系统

### Buff 类型

| 编号 | 名称 | 效果说明 |
|------|------|----------|
| buff1 | **力量** | 持续时间内每次使附着对象对其他人造成的物理伤害增加 n 点 |
| buff2 | **虚弱** | 持续时间内每次使附着对象对其他人造成的物理伤害减少 n 点 |
| buff3 | **抗性** | 持续时间内每次使附着对象受到的物理伤害减少 n 点 |
| buff4 | **易伤** | 持续时间内每次使附着对象受到的物理伤害增加 n 点 |
| buff5 | **治愈** | 持续时间内每回合回复附着对象 n 点血量 |
| buff6 | **凋零** | 每次附着对象回血时消耗 1 层凋零并减少 1 点回血 |
| buff7 | **护盾** | 每次附着对象受到物理/火焰伤害时消耗 1 层护盾减少 1 点伤害 |
| buff8 | **抗火** | 持续时间内使附着对象免疫火焰伤害 |
| buff9 | **中毒** | 每次附着对象回血后减少 2 点血量（每回合限 2 次） |
| buff10 | **蓄力** | 附着对象造成物理伤害时消耗全部蓄力点，伤害 + 等量力量，被伤害人 + 等量凋零 |
| buff11 | **治愈增强** | 持续时间内每次附着对象回血时多回 1 点血 |

### Buff 机制
- **限时状态**：持续 n 回合，持续时间结束后状态消失
- **层数机制**：效果附着在玩家身上，层数耗尽后状态消失
- **叠加规则**：同类型 buff 不合并，各自独立计算层数和持续时间
- **每回合结束时**：所有限时状态的剩余回合数 -1，若 ≤ 0 或强度/层数 ≤ 0 则移除

---

## 定义

| 术语 | 说明 |
|------|------|
| **限时状态** | 持续 n 回合数的状态，持续时间结束后状态消失 |
| **状态** | 附着在玩家身上的卡牌效果，效果层数耗尽后状态消失 |
| **丢弃** | 玩家可以随时扔掉手牌至废牌堆 |
| **手牌** | 玩家拥有的未生效或未装备的牌 |
| **向上取整** | 若数值存在小数部分，则向上取整数 |
| **向下取整** | 若数值存在小数部分，则向下取整数 |
| **装备槽** | 装备卡占用的槽位（equip/weapon/field 三个独立槽位） |

## 特殊交互

### 卡牌使用
- 卡牌可以对**任意对象**使用（自己或对手）
- 当卡牌拥有多个效果时，按效果顺序**依次生效**
- 当卡牌被舍弃时，触发相应的舍弃事件
- 装备卡在**装备时生效**，在轮到装备者自己的回合时刷新状态

### 牌组循环
- 牌组抽空后，自动将**弃牌堆**洗回牌组
- 保证对局不会因牌尽而中断

## 技术实现

### 代码结构
```
shared/           ← 核心游戏引擎（前后端共用）
  types.ts        - 类型定义
  constants.ts    - 常量 & 卡牌模板
  buffEngine.ts   - Buff 计算引擎（纯函数）
  cardEngine.ts   - 卡牌效果引擎（纯函数）
  gameEngine.ts   - 游戏流程引擎（纯函数）
  validation.ts   - 动作校验

server/           ← 服务端（Express + Socket.IO）
  src/
    index.ts      - 入口 & WebSocket 事件
    rooms.ts      - 房间管理 & 游戏调度

client/           ← 前端（React + Vite）
  src/
    pages/        - Lobby（大厅）、Game（对战）
    components/   - UI 组件
    hooks/        - Socket 连接
    store/        - Zustand 状态管理
```

### 关键架构原则
1. **服务端权威** — 所有游戏逻辑由服务端执行，客户端只做展示
2. **纯函数引擎** — gameEngine/cardEngine/buffEngine 均为纯函数，输入旧状态 → 输出新状态，不修改原对象
3. **事件驱动 Buff** — Buff 不直接修改数值，通过事件钩子介入计算流程（伤害计算、回血计算）
4. **状态过滤** — 发送给客户端时隐藏对手的手牌和牌库
